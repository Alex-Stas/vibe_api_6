# API_6: CLI-чат с OpenAI и Anthropic через ProxyAPI

Небольшой консольный чат на Python с поддержкой двух режимов:

- `thinking` (по умолчанию): Claude (Anthropic) с выводом шагов размышления.
- `normal`: OpenAI Chat Completions без блока размышлений.

В обоих режимах поддерживается диалог с сохранением контекста: история сообщений хранится в памяти и отправляется в каждый новый запрос.

## Структура проекта

- `chat_completions_test.py` - основной файл приложения.
- `requirements.txt` - зависимости Python.
- `.env` - переменные окружения (ключи, URL, модели, режим).

## Требования

- Python 3.10+ (проверено на Python 3.13).
- Доступ к ProxyAPI (`api.proxyapi.ru`) и валидный API-ключ.

## Установка

1. Создать и активировать виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
```

2. Установить зависимости:

```powershell
python -m pip install -r requirements.txt
```

3. Заполнить `.env` (пример):

```env
AI_API_KEY=your_key_here
OPENAI_BASE_URL=https://api.proxyapi.ru/openai/v1
OPENAI_MODEL=gpt-4o
ANTHROPIC_BASE_URL=https://api.proxyapi.ru/anthropic
ANTHROPIC_MODEL=claude-sonnet-4-6
CHAT_MODE=thinking
```

## Запуск

```powershell
python chat_completions_test.py
```

При старте программа спросит режим:

- `thinking` или `normal`
- если нажать Enter, будет взят режим из `CHAT_MODE` (по умолчанию `thinking`)

Команды в чате:

- `reset` - очистить историю диалога
- `exit` - завершить программу

## Описание функций и классов

### Функции

- `_read_api_key() -> str`  
  Читает `AI_API_KEY` из окружения. Если ключ отсутствует, выбрасывает `ValueError`.

- `create_openai_client() -> OpenAI`  
  Создает OpenAI-клиент для режима `normal`, используя `OPENAI_BASE_URL`.

- `create_anthropic_client() -> anthropic.Anthropic`  
  Создает Anthropic-клиент для режима `thinking`, используя `ANTHROPIC_BASE_URL`.

- `select_chat_mode() -> str`  
  Запрашивает режим у пользователя (`thinking`/`normal`), применяет значение по умолчанию из `CHAT_MODE`.

- `interactive_chat() -> None`  
  Основной цикл CLI-чата: инициализация выбранного режима, обработка `reset`/`exit`, отправка сообщений и вывод ответа.

### Классы

- `OpenAIChatSession`  
  Сессия диалога для OpenAI Chat Completions:
  - хранит историю в `self.messages`
  - `ask()` отправляет всю историю и сохраняет ответ
  - `reset()` очищает историю (опционально сохраняет `system` сообщение)
  - `get_history()` возвращает текущую историю

- `AnthropicThinkingSession`  
  Сессия диалога для Claude в режиме размышлений:
  - хранит историю в `self.messages`
  - `ask()` отправляет историю + `thinking` параметры, возвращает:
    - финальный ответ ассистента
    - список шагов размышлений (`thinking_steps`)
  - `reset()` очищает историю

## Обработка ошибок

Ниже перечислены основные ошибки и текущая реакция программы.

1. Отсутствует API-ключ (`AI_API_KEY`)

- Где возникает: `_read_api_key()`
- Тип: `ValueError`
- Реакция: программа завершится с сообщением  
  `Set AI_API_KEY before running this script.`

2. Неизвестный режим при вводе

- Где возникает: `select_chat_mode()`
- Тип: не исключение, штатная обработка
- Реакция: выводится предупреждение  
  `Unknown mode '...', using default: ...`  
  затем продолжается запуск с режимом по умолчанию.

3. Ошибки сети / таймаут / недоступность ProxyAPI

- Где возникает: при вызове API в `ask()`
- Тип: исключения SDK (`openai`/`anthropic`) или сетевые исключения
- Реакция: программа выводит понятное сообщение о сетевой ошибке, показывает детали исключения, затем ждет нажатия любой клавиши и завершает работу.

4. Неверный ключ / проблемы авторизации / неподдерживаемая модель

- Где возникает: при API-запросе в `ask()`
- Тип: исключения провайдера (401/403/404/422 и т.п.)
- Реакция: программа выводит сообщение о проблеме авторизации или модели, показывает детали исключения, затем ждет нажатия любой клавиши и завершает работу.

5. Нестандартное состояние инициализации сессии

- Где возникает: `interactive_chat()`, если сессия не создана
- Тип: `RuntimeError("Chat session was not initialized.")`
- Реакция: программа выводит сообщение об ошибке инициализации, показывает детали исключения, затем ждет нажатия любой клавиши и завершает работу.

## Рекомендации по безопасности

- Не храните реальные ключи в публичных репозиториях.
- Добавьте `.env` в `.gitignore`, если проект будет версионироваться.
- При компрометации ключа немедленно отзовите его и выпустите новый.
